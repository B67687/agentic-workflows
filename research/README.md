# Research

Track daily discoveries in AI agent engineering, token efficiency, and trending repos.

## Quick Start

Run research manually:
```text
Use the prompt in research-prompt.md to research what's new, then add findings to research-log.md.
```

## Structure

| File | Purpose |
|------|---------|
| `research-prompt.md` | Reusable prompt for daily research runs |
| `research-log.md` | Active research intake and campaign index |
| `../archive/research/research-findings.md` | Durable distilled discoveries worth keeping |
| `../archive/research-log-YYYY-MM.md` | Full completed research campaigns |

## What's Tracked

1. **Revolutionary agents** — self-improving, memory systems, learning loops
2. **Token efficiency** — prompt compression, KV cache, caching strategies  
3. **Trending repos** — GitHub AI/agent/engineering hot repos
4. **MCP ecosystem** — new servers, protocols, tools

## Workflow

1. Run research prompt (manual or scheduled)
2. Add active entry to `research-log.md`
3. When the campaign is complete, move the full long-form record to `archive/`
4. Add only durable reusable conclusions to `../archive/research/research-findings.md`
5. Leave a compact index row in `research-log.md`

## Retention Rule

Do not let `research-log.md` become the full archive. It should stay cheap to read on resume.

Preserve detailed completed campaigns in `archive/` rather than deleting them.
